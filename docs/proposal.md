## Reflection

This job posting is relevant to this course because it centers around using data to find and prioritize targets that align with the needs of the club. In the requirements of the job, the employee must build and maintain automated reports, based on raw data that has been ingested, interpreted, and transformed in order to measure game model efectiveness and prepare for upcoming match fixtures. The skills that the job requires, such as querying SQL databases, data visualization, and version control using Git, closely aline with the coursework, as the goal is to interpret player and match data and visualize it with clean and accurate recruitment and performance metrics. To prove I can do this job, I would build a pipeline that ingests MLS player and match data from a public API, transforms it through staged models into dimension tables focused on players, matches, and opposing teams, and presents the results in a dashboard with the precise metrics and recruitment scorecards that the techinal staff would presumably use. This same project can transfer into adjacent roles, including analytical positions at a sports media company such as ESPN or a data analyst for a sports team not in the MLS, perhaps the Los Angeles Clippers in the NBA. Data sources I would use for this project would include player and match data from the football-data.org API, as well as scraping player profile and season stats pages from FBref.com.
